Automatización Formato Unico de Notificación

Para el requerimiento único de noticia se quería estandarizar dinámicas y procesos que facilitan al personal gestionar el formato único de notificación, sin generar errores de llenado de información o que generaran reprocesos o conductas graves por una falsa notificación. Es por ello por lo que se creó un formato de llenado en una plantilla de Excel mediante la cual se pudiera estandarizar el orden de llenado para los campos requeridos y definir el formato de campos de fecha y texto para la conformación del documento en PDF.

Dentro de la visual de análisis de datos se crean visuales de manera automática mediante macros ligadas a los tres crudos disponibles (clientes, ventas y productos). Estos tres grupos son unificados mediante power pivot y generados de manera automática en la Hoja “Análisis” mediante la macro ligada al botón “Generador de Reporte”. Dentro de las visuales generadas para cada apartado, se generan gráficos de barras para comparar de manera trimestral el rendimiento de las ventas de los productos top que mas se han vendido y los tipos de clientes por genero y edad. Ademas de su correspondiente comparativa segun sus categorias y tipos de productos.

Para la creación de la interfaz de ingreso y las herramientas de interacción. Se crearon macros para la recolección de información de las celdas disponibles, se crea la macro de limpiar para eliminar los registros temporales en las celdas de datos, se crea la macro guardar información, la cual permite guardar los registros disponibles dentro de las celdas de datos, además de limpiar las celdas de datos y generar una notificación de guardado exitoso si las condiciones de llenado y de tipo de dato se cumplan.

La interfaz de ingreso de datos contiene visuales dinámicas que permiten generar vistas temporales según tabla dispuesta en la parte derecha. Estos datos seleccionados se presentan en cada uno de los espacios dispuesto al lado izquierdo de esta manera se permite generar una eliminación, guardado y modificación de los datos disponibles dentro de las bases o el ingreso de nuevos registros dentro de la base y categorizarlo en su respectiva base de datos.

Para el almacenamiento y guardado de información se realizó una macro que según los datos registrados en los recuadros se establecieran variables TextBox que tomaran los datos dispuestos en el recuadro y lo almacenen inicialmente en la base de datos clientes. Para de esta manera realizar un almacenamiento de los datos con todas sus generalidades y posteriormente realizar una categorización de manera automática de los valores de venta y el tipo de producto comprado según el registro único de compra del cliente en cuestión.

Una de las funciones primordiales de la aplicación y que se encuentra ligada de manera transversal a todas las herramientas de la interfaz inicial; es la función previsualización, la cual permite la previsualización de datos en las cajas de texto para el respectivo uso de las herramientas de guardado, limpieza, eliminación y modificación de datos. Para la creación de esta macro se tuvo presente una ruta definida dentro de las cajas de texto y un destino definido para el manejo de la data. Además para este proceso se destinó una variable que permitiera la interacción de las diferentes fuentes de datos y para ello se tomó la llave única de cada transacción para la búsqueda única y relativa de los datos según cliente.
