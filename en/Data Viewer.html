Data Viewer

Data viewer is a graphical interface that allows you to view SQL data in a decentralized way, with a graphical interface friendly and functional for basic operating activities. In which we can find: add new data, delete data, modify data, import a new database or export data in csv, xlsx and pdf format.

For the initial layouts of the interface, it was kept in mind that it should be an easy and functional interface. Where The priority was to have three main functions of frequent use and two complementary functions of moderate use. To do this, the interface functionalities were structured in three scripts; where it was established that, in the first script, the connection with the SQL database will be made and all the initial search functions will be generated. In it The second script provided the launcher of the interface and its subsequent linked scripts. In the last script The entire visual of the interface, the buttons, functions and complementary actions of the interface are structured.

The base used has different parameters, which in this case were taken into account to structure the interface logic. Within these components we find that the base has numerical data, text, characters, dates and unique product indicators. Which in this case allow each of the purchases to be identified.

The composition of the first script is centered on two libraries pymysql for connection and integration with SQL and plyer for generating notifications. To do this, first a function was defined that would allow the connection to the SQL database, Then the host, user, password and database name parameters were defined. Connection already made with the base, the query data is defined through a list of characters, which are linked to the function data_consultation, which returns the data entered in the database.

The function data_consultation initially reads the SQL database by selecting the database and its consequent selected table. Once the data has been read in its entirety, a successful notification of entry to the base is chained.

For the search, deletion, insertion and modification of data, similar input and output parameters were used to define each function. In the case of data search, the function search_data is created, which takes as a reference the Id of the table, that is, it takes as references the first column of the table and with it calls all the data that has an Id. In this case, since the Id is a unique value for each of the data, the information is not repeated within the search.

To complement the functionality of each mechanism created, general multiple-use functions are structured, which aim to integrate interactions initials or finals in a homogeneous way. A clear example of this is the cancel function, which is organized by an alert message with two possible responses to the action taken. One performs the action and the other denies the interaction. However, although different basic actions are carried out, in Both interactions include a sweep, a disabling of bars and an activation of buttons.

Among the most relevant functions we find deleting and creating a new record, these two functions have a similar interaction structure, but execution different. In the case of the delete function, two methods are carried out; where the first interacts with the selected data by reading the row and its respective columns. Once it is verified that data exists, an alert is generated with two possible responses to the action performed where one performs the action and the other he denies it. If the selection is affirmative, the second function checks the data to be deleted and purges the data in the database.

Understanding the logic used within the data_query function. The function modify_data is carried out, in this case two logics are carried out for the interaction and execution of the function. The first calls all the columns defined in the base, for this it is chained with the second function, which defines the parameters of the modify button and with this the data displayed in Data Viewer is taken and transcribed in the text bars of the interface. Once modified the changes are inserted into the same selected column and the data in the database is updated using the first function.
