PROCESS AUTOMATION

API for Real-Time Analysis of Ecopetrol Data

The development of the system began with the identification of the relevant data sources for extraction, focusing on Ecopetrol's financial information published on the Colombian Stock Exchange. Based on this identification, an automated workflow was designed using an RPA, which is responsible for collecting and storing the data in a SQL database.

To ensure real-time analysis, a processing system was implemented that allows the most relevant results to be viewed through a web API hosted on localhost. This integration facilitates continuous monitoring of key variables and optimizes decision-making based on updated information. The structure of the system ensures efficiency in data management and automation of the extraction and analysis process.

Historical analysis involves collecting and evaluating past data to identify patterns, trends, and anomalies. In this project, we will work with historical financial data from ECOPETROL, one of the leading companies in the Colombian stock market. This data includes closing prices, high and low prices, transaction volumes, and percentage and absolute changes. The analysis of this historical data will allow us to establish a context for the behavior of ECOPETROL shares, which is essential for any predictive analysis or investment decision making.

Data collection represents the starting point of the model. An RPA (Robotic Process Automation) was implemented to interact directly with the Colombian Stock Exchange (BVC) website. This automated approach was chosen due to the following reasons:

• Eliminate repetitive and manual tasks: Downloading financial data manually is error-prone and consumes valuable time, especially when data needs to be updated frequently.

• Process standardization: Using tools like pyautogui and webbrowser ensures that interaction with the website is consistent and replicable.

Visualizing the real-time data is done through a web API running on a local server (localhost). When the Flask application is started with the flask run command, the console displays messages indicating the status of the server. The message "Serving Flask app 'app'" means that Flask has identified the application being run, in this case, named 'app'. The notice "Debug mode: off" indicates that debug mode is disabled, which is suitable for a development environment, but not recommended for production. The warning "WARNING: This is a development server. Do not use it in a production deployment. Use a production WSGI server instead" underlines that Flask is running on its built-in development server, which is suitable for testing and development, but not robust enough to handle a production deployment. For production, it is recommended to use a WSGI server, such as Gunicorn or uWSGI, which are more secure and scalable.

The proposed implementation of the RPA system for the extraction of Ecopetrol data from the Colombian Stock Exchange could be an effective tool to optimize operational efficiency in the management of financial information. The use of RPA would potentially eliminate the need to perform repetitive and error-prone tasks, such as manual data collection, allowing these activities to be executed automatically and without human intervention. This approach would not only streamline the workflow, but would also significantly reduce the risk of errors and variability in the quality of the data collected, essential aspects in a financial environment where accuracy and consistency are essential. Reducing human intervention would free up resources that could be redirected to analysis and strategic decision-making activities.
