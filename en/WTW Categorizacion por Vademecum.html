WTW / Categorization by Vademecum

Vademecum Categorization is a tool designed in Excel using Visual Basic for Applications, which aims, through different macros, to import data into the book, categorize the vademecum, validate the different vademecums according to their application or non-application, previews using forms for the visualization of data that apply according to the vademecum and an export of the data that applies within a new sheet.

For the analysis of crude oil data, agile mechanisms were established that would allow for an analysis of several documents, keeping in mind that there is a base document where the crude oil would be integrated with the vademecum analysis and on the other hand, three different types of documents would be connected. , but with a causal relationship with the unique identifier of the base document. Keeping in mind that for this purpose, a macro was designed that will extract the three raw materials and integrate them into the base document in order to begin the second phase of the document, which is the analysis and differentiation of the vademecum.

To combine data from multiple Excel sources into a single file using BVA (Business Value Added), first, you connect the Excel data sources to the BVA platform through its interface. Data is then imported from each source into BVA and processed using the data manipulation tools available on the platform to merge, clean and transform the data sets as necessary. Finally, the combined data is exported to a new Excel file, facilitating efficient consolidation and analysis of information from various sources.

Once the information is collected, the vademecum categorization is carried out by automating the search and comparison of codes between two spreadsheets within an Excel workbook. First, variables are declared to store information, such as the line counter, the last row of data in the "WTW" sheet, and the code variables to search for. Then, the search range is set on sheet "3". Next, a loop is started that goes through the rows of the "WTW" sheet starting from row 8 to the last row with data. In each iteration of the loop, the code is taken from column C of sheet "WTW", its correspondence is searched in the range defined in sheet "3" using Excel's VLookup function, and the result is assigned to the variable "code2". If the search returns no results (that is, returns an error), the value "Does not match" is assigned to "code2"; otherwise, "Cross" is assigned. Subsequently, the result of the comparison is written in column 13 of the "WTW" sheet.

This formula code in Excel is used to perform a conditional evaluation based on multiple conditions. Using the IF function and the AND function, first check if the value of the "Belongs to vademecum" column is "Not Vademecum", then check if the value of the "Vademecum1" column is "Cross", and finally, if the value of the "Date grouping" column is "Vademecum 1". If all of these conditions are true, the formula returns "Find", indicating that a significant match has been found. Otherwise, it returns "Not Applicable". This formula allows you to automate the data evaluation process according to multiple criteria, facilitating the identification of relevant findings in a data set.

Once all the data validations were completed, an interaction interface was created using an Excel form that would allow the three Vademecum to be categorized according to two variables “Findings” and “Not applicable.” Once one of the two options has been defined according to the Vademecum, the data can be exported according to each element of the list described in the corresponding cells of the "Report" worksheet, starting from the row indicated by "initial". After writing the values, the variable "uf" is incremented by 1 to move to the next row. This code is used to export data from a UserForm to an Excel sheet in an orderly and automated manner.

It is used to remove duplicate records in an Excel sheet. First, a variable called "value" is declared as a Long type. The code then selects cell A5 on the "Report" sheet. A "Do While" loop is then started and runs as long as the active cell is not empty. Inside the loop, Excel's "CountIf" function is used to count how many times the value of the active cell appears in column A. If the value is greater than 1 (that is, if there is more than one occurrence), it is removed the entire row containing the active cell. If the value is 1 (that is, if the record is unique), the selection is moved to the next row using "Offset". This process is repeated until duplicate records are found throughout column A. Finally, the code selects cell A5 to ensure that the process starts from the beginning the next time it runs. In short, this code removes duplicate records in an Excel sheet, keeping only one instance of each unique record.
